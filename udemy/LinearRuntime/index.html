Linear Data Structures

LinkedList

A data structure that holds a head, tails, and length property. They consist of nodes, and each nodeshas a value and a pointer to another node or null.

  • Singly Linked List are an excellent alternative to arrays when insertion and deletion at the beginning are frequently required.
  • Arrays contain a built-in index whereas Linked Lists do not.
  • The idea of a list data structure that consists of nodes is the foundation for other data structures like stacks and queues.

      • Big O Scale:

    • insertion: O(1)
    • Removal: O(1) or O(N)
    • searching: O(N)
    • Access: O(N)

Dounbly Linked List

Very similar to a singly linked list and it contains two points a previous and a next.

  • Doubly Linked List are almost idential to Singly Linked List except there is an additional pointer to the previous nodes.
  • Better than Singly Linked List for finding nodes and can be done in half the time.
  • However they do take up more memory because of the extra pointer associated in a Doubly Linked List.

      • Big O Scale:

    • insertion: O(1)
    • Removal: O(1)
    • searching: O(N) or O(N/2) <-- still O(N)
    • Access: O(N)

Stacks

They're data colloection and abide to the LIFO: Last In First Out Every stack has a size that determines how many nodes it can accomodate. Basically storing data in such a way where the first thing added is the last thing removed

There is more than one way to implement a stack.

For an array, it's very easy. Pushing the nodes inside then shifting them.

Building them as a class is a similar structure to Linked List in terms of push, pop, shift, and unshift

    Examples of using FIFO

  • Managing functions invocations.
  • Undo/Redo
  • Routing (the history object) is treated like a stack

      • Stack data structure has three main methods:

    • push(): adds a node to the top of the stack
    • pop(): removes a node from the top of the stack
    • peek(): returns the value of the top node without removing it from the stack

      Big O Scale:

    • insertion: O(1)
    • Removal: O(1)
    • searching: O(N)
    • Access: O(N)

Queue

They're data collection and abide to the FIFO: First In First Out structure. Very similar to a stack and there is also many different ways to implement them. (Class or arrays)

For an array, were unshifting the items and then pop them.

Building them as a class is a similar structure to Linked List in terms of push, pop, shift, and unshift

    Examples of using FIFO in programming

  • Waiting in line for shoes
  • For background tasks
  • Printing/Task processing

      • Queues provide three methods for interaction:

    • Enqueue: adds data to the “back” or end of the queue
    • Dequeue: provides and removes data from the “front” or beginning of the queue
    • Peek: reveals data from the “front” of the queue without removing it

      Big O Scale:

    • insertion: O(1)
    • Removal: O(1)
    • searching: O(N)
    • Access: O(N)
